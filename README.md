# Sweet-neighbors-club

Сервис для подбора недвижимости
---

## Описание проекта

Современный рынок недвижимости характеризуется высокой динамикой: ликвидные объекты могут быть сняты с публикации в течение нескольких часов. Потенциальные покупатели и арендаторы вынуждены мониторить несколько крупных агрегаторов («Авито», «Циан», «Домклик») одновременно, что приводит к информационной перегрузке и риску пропуска выгодных предложений.

**Сервис для подбора недвижимости** — это информационная система, которая автоматизирует процесс сбора, обработки и доставки пользователям релевантных предложений с популярных веб-ресурсов. Система объединяет данные из нескольких источников, предоставляет расширенные возможности фильтрации и оперативно уведомляет пользователя о новых объявлениях.

### Проблема
- Необходимость ручного просмотра нескольких сайтов.
- Информационная перегрузка из-за большого количества нерелевантных объявлений.
- Высокий риск пропуска выгодного предложения из-за отсутствия мгновенных уведомлений.
- Ограниченные возможности фильтрации на отдельных площадках.

### Решение
- Объединение данных из трёх основных источников в едином интерфейсе.
- Единые параметры фильтрации для всех источников.
- Централизованное управление уведомлениями (Push/Email).
- Поддержка PWA для доступа с мобильных устройств без установки нативного приложения.

---

## Ключевые возможности

- **Агрегация источников:** Сбор данных с «Авито», «Циан», «Домклик».
- **Умная фильтрация:** Поиск по цене, площади, району, типу дома, этажу, году постройки и др.
- **Мониторинг в реальном времени:** Обновление данных не реже одного раза в 10 минут.
- **Мультиканальные уведомления:** Настройка каналов связи (Push/Email) под потребности пользователя.
- **Безопасность:** Регистрация, аутентификация и авторизация пользователей.
- **Финансовая модель:** Freemium (неделя бесплатно, далее подписка).

## Технологический стек

| Компонент | Технология | Обоснование выбора |
| :--- | :--- | :--- |
| **Backend** | `Python` + `FastAPI` | Высокая производительность, асинхронность, автогенерация OpenAPI |
| **Frontend** | `React` | Динамический пользовательский интерфейс, компонентный подход |
| **Database** | `PostgreSQL` | Надежная реляционная СУБД, поддержка сложных запросов и транзакций |
| **Task Queue** | `Celery` | Фоновые задачи (скрапинг, уведомления), распределенная обработка |
| **Cache/Broker** | `Redis` | Брокер сообщений для Celery, кэширование часто запрашиваемых данных |
| **Mobile** | `PWA` | Progressive Web App, push-уведомления без нативной разработки |
